Bushido Secure Loadsheet Confirmation Platform

Rol: Full-Stack Software Engineering Intern Organisatie: KLM Royal Dutch Airlines Periode: Jaar 2 Semester 2 (2025) Type: Stageproject + Onderzoeksproject Domeinen: Enterprise IT, Aviation, Security & Compliance

Over het Project

Dit project is uitgevoerd tijdens mijn stage bij KLM Aircraft Turnaround. Het richtte zich op een kritiek probleem binnen de wet lease-operaties: de handmatige bevestiging van loadsheets (essentiële documenten voor gewichts- en balansverdeling in de luchtvaart).

Het probleem was dat loadsheets handmatig werden bevestigd zonder goede IT infrastructuur, wat leidde tot significante security, compliance en traceability risico's. Er was geen audit trail, onvoldoende toegangscontrole, en geen digitale handtekening mogelijk.

Als oplossing heb ik zowel een werkende digitale oplossing (Bushido) gebouwd als een onderzoeksrapport geschreven over deze risico's. Het Bushido platform digitaliseert het proces, biedt volledige traceability, en implementeert security best practices.

Bushido Technische Oplossing

Architectuur: Bushido is gebouwd als een enterprise grade full-stack applicatie met een event driven architectuur. De backend is ontwikkeld in Java met Spring Boot, de frontend in Angular, en voor dataopslag wordt Azure Table Storage gebruikt.

Enterprise Integraties: Het platform integreert met verschillende enterprise systemen via SOAP-API's (Amadeus via SOARepo) en enterprise event queues voor real-time updates. De applicatie draait in een enterprise test environment met een volledige CI/CD pipeline.

Belangrijke Functionaliteiten: Het platform implementeert een event driven workflow waarbij WB-events worden verwerkt. Loadsheets worden automatisch opgehaald via SOAP, piloten kunnen via de frontend accepteren of weigeren met volledige audit logging (wie, wat, wanneer). Het systeem beheert versiebeheer van FEAs en verwijdert automatisch rejected entries na 45 minuten via een scheduler.

Workflow Technisch Proces

Het Bushido platform volgt een gestructureerd, event driven proces:

  1. WB-event komt binnen via enterprise event queue
  2. Backend controleert Azure Table Storage voor bestaande entries
  3. Entry wordt aangemaakt of geüpdatet
  4. Loadsheet wordt opgehaald via SOAP-API's (Amadeus via SOARepo)
  5. Frontend toont loadsheet aan gebruiker
  6. Gebruiker accepteert of weigert via interface
  7. Status + timestamp worden opgeslagen met volledige audit informatie
  8. Event wordt gepubliceerd naar enterprise event queue
  9. Scheduler verwijdert rejected entries automatisch na 45 minuten

Deze workflow zorgt voor volledige traceability: elke actie wordt gelogd, elke loadsheet heeft een duidelijke status, en het systeem is betrouwbaar en schaalbaar voor enterprise gebruik.

Security Research & Analyse

Naast de technische implementatie heb ik uitgebreid onderzoek gedaan naar de security risks bij loadsheetbevestiging, gebruikmakend van erkende frameworks: NIST, OWASP, EASA en IATA standaarden. Het onderzoek combineerde literatuuronderzoek met interviews met de Product Owner en praktijkervaring als loadcontroller.

Belangrijkste Bevindingen: Het onderzoek identificeerde kritieke security en compliance risico's: geen audit trail, onvoldoende toegangscontrole, geen digitale handtekening, en geen offline-proof oplossing.
