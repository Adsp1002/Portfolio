Election Platform

Rol: Full-Stack Developer (Security & Deployment focus) Opleiding: HBO-ICT Software Engineering Jaar / Blok: Jaar 2 Blok 1 Type: Teamproject (SCRUM) Frontend: Vue.js Backend: Java (Spring Boot) Security: JWT Authentication Deployment: Docker + Render

Context van het Project

Het doel van dit project was het bouwen van een verkiezingsplatform waarmee gebruikers verkiezingsdata konden bekijken, politieke informatie konden lezen, en (na inloggen) interactie konden hebben. Dit project was onderdeel van Jaar 2, Blok 1 (Semester 3) van de HBO-ICT opleiding en viel onder het Project Election.

De applicatie bestond uit een Vue.js frontend, een Java Spring Boot backend, met beveiligde communicatie via JWT tokens. Het was een teamproject waarbij we volgens SCRUM-methodologie werkten, waarbij ik me focuste op security en deployment.

De applicatie moest gedeployed worden en publiek toegankelijk zijn. Dit betekende dat ik niet alleen moest zorgen voor werkende functionaliteit, maar ook voor een stabiele, beveiligde en beschikbare applicatie die 24/7 online toegankelijk was.

Mijn Concrete Bijdrage

Security & Authenticatie (kernbijdrage)

Mijn belangrijkste bijdrage lag bij security en authenticatie. Ik heb het volledige login- en registratiesysteem geïmplementeerd met JWT-authenticatie. Dit was mijn eerste ervaring met enterprise-achtige security implementatie.

JWT-authenticatie opgezet in de backend: Ik implementeerde JWT (JSON Web Token) authenticatie in de Java Spring Boot backend. Dit omvatte:

  • Token generation bij login: Wanneer een gebruiker succesvol inlogde, genereerde het systeem een JWT token met user claims. Dit token bevatte informatie over de gebruiker en werd gebruikt voor authenticatie bij volgende requests.
  • Token validatie via filters: Ik implementeerde Spring Security filters die JWT tokens valideerden bij elke request. Deze filters controleerden of het token geldig was, niet verlopen was, en of de gebruiker toegang had tot de gevraagde endpoint.
  • Beveiliging van API-endpoints: Ik beveiligde specifieke endpoints zodat alleen geauthenticeerde gebruikers er toegang toe hadden. Dit betekende dat bepaalde functionaliteit, zoals het plaatsen van reacties of het bewerken van profielen, alleen beschikbaar was na succesvolle authenticatie.

Frontend en backend veilig laten communiceren: Ik zorgde ervoor dat de Vue.js frontend en Java backend veilig met elkaar communiceerden via JWT headers. Wanneer een gebruiker inlogde, ontving de frontend een JWT token dat werd opgeslagen. Bij elke volgende request naar beveiligde endpoints werd dit token meegestuurd in de Authorization header. De backend valideerde dit token voordat de request werd verwerkt.

Correct scheiden van publieke en beveiligde endpoints: Ik was verantwoordelijk voor het correct scheiden van publieke endpoints (zoals het bekijken van verkiezingsdata en politiek nieuws) en beveiligde endpoints (zoals het plaatsen van reacties of het bewerken van profielen). Publieke endpoints waren toegankelijk zonder authenticatie, terwijl beveiligde endpoints JWT validatie vereisten.

Backend (Java / Spring Boot)

Ik heb backend-logica geschreven voor authenticatie, gebruikersbeheer en API-koppelingen. Dit omvatte het opzetten van controllers en services volgens een gelaagde architectuur.

Controllers en services opgezet: Ik bouwde controllers die HTTP requests ontvingen en services die de business logic bevatten. Deze gelaagde structuur zorgde voor scheiding van concerns: controllers handelden HTTP-specifieke zaken af, terwijl services de business logica bevatten. Dit maakte de code modulair en testbaar.

Meewerken aan databewerking en veilige datatoegang: Ik werkte aan databewerking en zorgde voor veilige datatoegang. Dit betekende dat ik database queries schreef die gebruikersinformatie ophaalden en opsloegen, terwijl ik ervoor zorgde dat gevoelige data niet onnodig werd blootgesteld. Ik implementeerde ook input validatie om te voorkomen dat ongeldige of kwaadaardige data in de database terechtkwam.

Backend geschikt maken voor containerized deployment: Ik maakte de backend geschikt voor containerized deployment met Docker. Dit betekende dat ik configuratie aanpaste zodat de applicatie in een container kon draaien, environment variables gebruikte voor configuratie, en ervoor zorgde dat de applicatie goed startte in een containeromgeving.

Frontend (Vue.js)

Aan de frontend-kant heb ik de hoofdpagina van de applicatie ontworpen en gebouwd, en politiek nieuws opgehaald via een externe API en dynamisch weergegeven.

Hoofdpagina ontworpen en gebouwd: Ik bouwde de hoofdpagina die als startpunt diende voor gebruikers. Deze pagina toonde belangrijke informatie zoals verkiezingsdata, politiek nieuws, en navigatie naar andere delen van de applicatie. Ik gebruikte Vue.js componenten om de pagina modulair op te bouwen.

Politiek nieuws ophalen en dynamisch weergegeven: Ik implementeerde functionaliteit om politiek nieuws op te halen via een externe API en dit dynamisch weer te geven in de applicatie. Dit betekende dat ik API calls maakte vanuit de Vue.js frontend, de data verwerkte, en deze presenteerde aan gebruikers in een gebruiksvriendelijke interface.

Vue-concepten toegepast: Ik paste Vue-concepten toe zoals:

  • Component-structuur: Ik bouwde herbruikbare componenten voor verschillende delen van de interface, zoals nieuws items, gebruikersprofielen, en navigatie-elementen.
  • API-calls: Ik maakte API calls naar de backend en externe APIs om data op te halen en te versturen. Ik gebruikte async/await voor het afhandelen van asynchrone operaties.
  • State-afhandeling voor gebruikersstatus: Ik beheerde de state van de applicatie, met name de gebruikersstatus (ingelogd/niet ingelogd). Wanneer een gebruiker inlogde, werd de state bijgewerkt en de UI aangepast om de juiste functionaliteit te tonen.

Frontend-logica correct laten samenwerken met JWT-beveiligde endpoints: Ik zorgde ervoor dat de frontend-logica correct samenwerkte met JWT-beveiligde backend-endpoints. Dit betekende dat ik JWT tokens opsloeg na login, deze tokens meegestuurd bij requests naar beveiligde endpoints, en de UI aanpaste op basis van authenticatiestatus.

Deployment & Infrastructuur

Een belangrijk onderdeel van mijn bijdrage was deployment en infrastructuur. Ik heb de applicatie gedeployed met Docker en geschikt gemaakt voor Render.

Applicatie gedeployed met Docker: Ik containerized zowel de frontend als backend met Docker. Dit betekende dat ik Dockerfiles schreef die de applicaties pakketten, dependencies installeerden, en de applicaties startten. Docker zorgde ervoor dat de applicatie reproduceerbaar was en op verschillende omgevingen kon draaien.

Zowel frontend als backend geschikt maken voor Render: Ik maakte beide delen van de applicatie geschikt voor deployment op Render, een cloud hosting platform. Dit betekende dat ik configuratie aanpaste zodat de applicaties correct startten op Render, environment variables gebruikte voor configuratie, en ervoor zorgde dat de frontend en backend correct met elkaar communiceerden via Render's netwerk.

Dockerfiles en configuratie opgezet: Ik schreef Dockerfiles en configuratie zodat de applicatie reproduceerbaar was en online toegankelijk bleef. Dit betekende dat ik ervoor zorgde dat de applicatie automatisch startte wanneer de container draaide, gezondheidschecks implementeerde, en logging configureerde voor monitoring.

Praktische ervaring met cloud deployment en containerization: Dit gaf me praktische ervaring met cloud deployment en containerization. Ik leerde hoe je applicaties containerized, hoe je deze deployt naar cloud platforms, en hoe je ervoor zorgt dat applicaties stabiel en beschikbaar blijven. Dit was een belangrijke stap in mijn ontwikkeling als software engineer.

Resultaat

Het platform draaide succesvol als live webapplicatie. Gebruikers konden registreren en inloggen, beveiligde functionaliteit gebruiken, en actuele politieke informatie bekijken. De applicatie was publiek toegankelijk en voldeed aan de kernvereisten van security, full-stack integratie en deployment.

Functionaliteit: Het platform bood volledige functionaliteit: gebruikers konden accounts aanmaken, inloggen met JWT-authenticatie, verkiezingsdata bekijken, politiek nieuws lezen, en (na authenticatie) interactie hebben met het platform. Alle functionaliteit was beveiligd waar nodig en publiek toegankelijk waar mogelijk.

Security: De applicatie implementeerde enterprise-achtige security met JWT-authenticatie. Gebruikers konden veilig inloggen, en beveiligde endpoints waren beschermd tegen ongeautoriseerde toegang. Dit toonde aan dat ik in staat was om security te implementeren op een professioneel niveau.

Deployment: De applicatie was succesvol gedeployed en bleef online toegankelijk. Zowel frontend als backend draaiden stabiel in containers op Render, wat toonde dat ik in staat was om applicaties productierijp te maken en te deployen.

Wat ik heb Geleerd

Dit project was mijn eerste ervaring met enterprise-achtige security implementatie met JWT. Ik leerde hoe authenticatie en autorisatie werken in praktische applicaties, en hoe je security implementeert die niet alleen werkt, maar ook schaalbaar en onderhoudbaar is.

Begrip van authenticatie & autorisatie: Ik ontwikkelde diepgaand begrip van authenticatie (wie ben je?) en autorisatie (wat mag je?). Ik leerde hoe JWT tokens werken, hoe je ze genereert, valideert en gebruikt, en hoe je endpoints beveiligt op basis van authenticatiestatus.

Frontend-backend security flows: Ik leerde hoe security flows werken tussen frontend en backend. Ik begrijp nu hoe tokens worden uitgewisseld, hoe ze worden opgeslagen, en hoe ze worden gebruikt voor authenticatie bij requests. Dit begrip is essentieel voor het bouwen van beveiligde applicaties.

Deployment pipelines: Ik leerde hoe deployment pipelines werken en hoe je applicaties productierijp maakt. Ik begrijp nu hoe je applicaties containerized, hoe je ze deployt naar cloud platforms, en hoe je ervoor zorgt dat ze stabiel en beschikbaar blijven.

Grote stappen in Java, Vue.js en cloud-infrastructuur: Dit project betekende grote stappen voorwaarts in mijn vaardigheden met Java, Vue.js en cloud-infrastructuur. Ik leerde niet alleen de technologieën kennen, maar ook hoe je ze inzet voor realistische, productierijpe applicaties.

Hoe je een applicatie productierijp maakt: Ik leerde hoe belangrijk het is om niet alleen werkende code te schrijven, maar ook om te denken over deployment, security, monitoring en onderhoudbaarheid. Een applicatie die lokaal werkt is niet hetzelfde als een applicatie die productierijp is.
